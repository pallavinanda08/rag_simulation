RAG Interactive Learning Playground

1. Knowledge Base Input

Paste documents to simulate a knowledge base used in Retrieval Augmented Generation.

2. Chunking

Chunking splits large documents into smaller pieces so embeddings capture semantic meaning.

3. Embedding Generation

Embeddings convert text chunks into numeric vectors representing semantic meaning.

4. Vector Database

Vectors are stored in a vector database enabling fast similarity search.

5. User Query

Ask a question which will be embedded and compared against stored vectors.

6. Similarity Search

Cosine similarity compares the query embedding with stored vectors to retrieve relevant chunks.

7. Context Injection

Retrieved chunks become contextual knowledge passed into the LLM prompt.

8. Augmented Prompt

The user query and retrieved context are combined before sending to the LLM.

9. Generated Answer

The LLM uses the augmented prompt to produce an answer grounded in retrieved context.
